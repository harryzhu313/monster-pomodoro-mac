# Tomato Monster (macOS) · PRD · v1-launch

> 文件位置:`iterations/v1-launch/PRD.md`
> 配合项目根的 BRIEF.md 阅读。这份 PRD 描述本次迭代(v1)要做的事。

## 1. 项目背景

把 Chrome 插件版番茄钟(Tomato Monster · 按时停下来)重做为 macOS 菜单栏应用。产品哲学与视觉语言继承插件版:按时停下来、加时有代价、奖励"停下来"。旧插件代码位于本仓库同级目录 `../tomato-timer-v2`(本地私有仓库),可作功能与交互的参考基准。

参见:[BRIEF.md](../../BRIEF.md)

## 2. 本次目标

v1 是"功能对齐迁移 + 菜单栏形态升级":

1. 插件版已有功能全部平移(计时、配额、任务、收获、连击、热力图、主题、声音、Notion 导出、本地备份)。
2. 新增菜单栏形态的必要能力:菜单栏常驻倒计时、全局快捷键、开机自启。
3. 历史数据从插件的备份 JSON 一次性导入,导入后停用插件。

上线标准:作者的日常番茄完全脱离浏览器,在新 app 里闭环。

**重要决定**:强制锁屏在 v1 搁置(理由:处理紧急事务时被突然锁全屏体验有敌意)。到点干预降级为"强提醒 + 配额机制",详见 §8 风险第一条。

## 3. 用户与关键场景

### 场景 1:开始一天第一个番茄
- **用户**:作者本人,正在任意软件里工作(不一定开着浏览器)。
- **旅程**:按全局快捷键,或点菜单栏图标弹出主面板 → 添加/选中今日任务 → 点"开始专注" → 菜单栏出现倒计时。
- **结果**:不碰浏览器,3 秒内进入专注。

### 场景 2:番茄到点被提醒停下
- **旅程**:25 分钟到 → 提示音 + 主面板自动弹出并钉住(点击任何地方都收不回去,但不抢键盘焦点)+ 菜单栏变红进入休息倒计时,白噪音响起 → 起身休息 → 休息结束面板自动收起,自动开始下一个番茄(可在设置关闭)。
- **结果**:按时停下来,收获一颗 🍅,连击与热力图记一笔。

### 场景 3:忍不住想继续
- **旅程**:休息期间想接着干活 → 打开主面板点"加时" → 扣当日配额(每天 3 次)→ 回到专注。
- **结果**:续杯有代价、有记录;配额用完只能等休息结束。休息期间没有免费跳过出口。

### 场景 4:回顾与沉淀
- **旅程**:一天结束打开统计与设置窗口 → 看热力图、最近 7 天图表、历史明细 → 把当天任务导入 Notion。
- **结果**:专注记录沉淀在自己的 Notion 数据库里。

### 场景 5:从旧插件迁移
- **用户**:作者本人,首次启动新 app。
- **旅程**:在旧插件 options 页导出备份 JSON → 新 app 设置里点"导入备份" → 历史、连击、热力图数据延续 → 停用 Chrome 插件。
- **结果**:数据无缝衔接,一次性切换。

## 4. 界面结构

桌面应用没有路由,对应物是 6 个交互面(surface):

- **菜单栏图标区** —— 常驻入口 + 状态显示(倒计时文字、阶段状态)
- **主面板** —— 点菜单栏图标弹出,日常操作闭环(计时 / 任务 / 收获)
- **统计与设置窗口** —— 独立窗口,数据回顾 + 全部设置
- **休息提醒** —— 到点时的干预组合(声音 + 面板自动弹出钉住 + 菜单栏变红;系统通知已移除,2026-07-17 决策)
- **庆祝弹层** —— 7 天连击达成时的 love monster
- **全局快捷键** —— 无界面,在任何软件里开始 / 暂停专注

## 5. 信息架构

### 菜单栏图标区
- **状态图标** —— 随阶段切换。内容槽:图标 ×3(idle / 专注 / 休息,可用小怪兽表情 happy / calm / angry)
- **倒计时文字** —— 专注 / 休息剩余时间(mm:ss);休息期间整体变红醒目

### 主面板(弹出)
- **计时卡** —— 当前阶段与核心操作。内容槽:小怪兽表情 / 阶段文案 / 倒计时 / 主按钮(开始·暂停)/ 放弃(记烂番茄)/ 重置 / 今日配额(剩 N/3)/ 加时按钮(仅休息中可见,扣配额)
- **今日收获** —— 今天的成果一眼可见。内容槽:🍅 成熟 / 🥫 烂番茄图标行 / 连击天数(🔥 已连续 N 天)
- **今日任务** —— 任务管理。内容槽:添加表单(任务名 / 分类:工作·学习·生活·兴趣爱好 / 计划番茄数)/ 任务列表(当前专注标记、每任务完成计数)
- **底栏** —— 内容槽:打开统计与设置窗口按钮 / 随机激励提示语(1 句)

### 统计与设置窗口(分区平移旧插件 options 页)
- **7 天连击徽章** —— 本周连击进度与徽章位
- **年度热力图** —— 仿 GitHub 52 周热力图 + 月份标尺
- **最近 7 天** —— 柱状图 + 统计卡(当前连击 / 最长连击 / 总番茄)+ 清零今日
- **历史明细** —— 按日展开的任务记录,每日一个"导入 Notion"动作
- **本地备份** —— 导出 / 导入 JSON;导入兼容 Chrome 插件版备份格式(迁移通道)
- **Notion 导出配置** —— Token / 任务 DB ID / 日页面 DB ID(可选)/ 测试按钮 / 状态提示
- **番茄节奏** —— 长休息开关 / 每 N 个番茄(默认 4)/ 长休息时长(默认 20 分钟)
- **提醒与声音** —— 提示音开关 / 白噪音开关 / 最后一分钟提示
- **通用** —— 主题(default / monster,**默认 monster**)/ 开机自启 / 全局快捷键(可自定义)/ 休息结束自动开始下一番茄

### 休息提醒(到点干预)
- chime 提示音 → **主面板自动弹出并钉住**(失焦不隐藏、点托盘不收起、不抢键盘焦点)→ 菜单栏变红显示休息倒计时
- 休息结束面板自动收起(自动 / 手动模式皆同);加时成功也收起(选择了继续工作)
- 白噪音伴随整个休息期(可关,带淡入淡出)
- 休息期间主面板只有"加时(扣配额)"一个出口,无跳过按钮
- 已知限制:其他 app 的全屏 Space 上面板可能不可见,靠白噪音 + chime + 菜单栏 🔴 倒计时兜底

### 庆祝弹层
- 第 7 天连击的第一个番茄结束进入休息时,love monster 全屏庆祝,点击即关(是奖励,不是阻断)

## 6. 目标衡量

个人工具,指标以"自己真的用起来"为准,全部可从 app 本地统计观测:

**主指标(上线后 30 天):**
- 作者日常番茄 100% 走新 app,Chrome 插件停用卸载
- 计时漂移、崩溃 0 次(常驻可靠性)

**过程指标:**
- 工作日每天通过菜单栏 / 快捷键开始 ≥1 个番茄
- 番茄到点后正常进入休息的比例(而非中途放弃)
- 加时日均 ≤3 次(配额机制被正常使用而非绕过)
- 每周 ≥1 次打开统计窗口或执行 Notion 导出
- 迁移日:备份导入一次成功,历史连击 / 热力图数据完整

## 7. 范围

**这一版会做:** 核心计时(25/5 + 长休息)、菜单栏图标与倒计时、主面板(计时 / 任务 / 收获 / 配额)、统计与设置窗口(连击 / 热力图 / 7 天 / 历史 / 备份 / Notion / 全部设置)、休息强提醒 + 白噪音 / 提示音、小怪兽情绪主题、全局快捷键、开机自启、插件备份 JSON 导入。

**这一版不做:** 强制锁屏(v2 重新评估,候选方案"缓冲式锁屏":到点先横幅倒数 60 秒、期间可扣配额豁免、倒数完才锁)、"结束要说出口"(元认知输入)、日末自动提醒、手机端 / 手机同步、桌面悬浮窗(被菜单栏取代)、上架与分发投入、账号体系、Windows / Linux。

## 8. 风险与待解决问题

**已知项(请使用中持续确认):**

- **产品**:不锁屏后,"忽略提醒继续干活"没有实际阻断,强制性只剩提醒 + 配额心理账户。v1 用满一段时间后评估 v2 是否上"缓冲式锁屏"。(2026-07-17 部分缓解:休息时面板自动弹出且点击无法收起,介于"可忽略的通知"与"全屏锁"之间)
- **技术**:菜单栏每秒刷新倒计时文字的功耗与视觉抖动需验证;有问题降级为 30 秒 / 整分刷新。
- **技术**:WKWebView 的自动播放策略可能拦截白噪音 / 提示音,需早期验证;必要时改由 Rust 侧播放音频。
- **技术**:Notion API 有 CORS 限制,浏览器侧 fetch 会被拦,需从 Tauri 的 Rust 侧(HTTP 插件)发请求。
- **数据**:旧插件备份 JSON 是导入 schema 的基准,导入要有校验与失败提示,防止半截导入污染数据。
- **系统**:开机自启授权、全局快捷键与其他软件冲突(必须可自定义)——首次启动需要引导流程。(通知权限已随系统通知移除,不再需要)
- **分发**:自用不做签名公证;首次打开需右键 → 打开绕过 Gatekeeper,README 写清楚即可。
